Executive Summary

エグゼクティブサマリー

  • 振り子が「個性」側に振れ始めた。2024年までの主流だった「フラットで均質なミニマリズム+ベントグリッド+ダーク×グラデーション」は完全に定着した結果、2025年後半から「どこも同じ見た目」への反動——ネオブルータリズム、マキシマリズム、ノイズ・グレインなどの質感表現、手描き風——が明確なトレンドとして立ち上がっている。
  • 2025年はプラットフォーマーの刷新の年。Appleは6月のWWDCで「Liquid Glass」(iOS 26等に導入、iOS 7以来最大級のUI刷新)を、Googleは5月に「Material 3 Expressive」を発表。両者とも均質化への対抗としての「表現性」を掲げており、ガラス質感と弾むモーションがWebにも波及している。
  • 見た目の裏で「ルール化」が進んだ。WCAG 2.2(2023年勧告)、欧州アクセシビリティ法(EAA、2025年6月28日適用開始・すでに訴訟事例あり)、Core Web VitalsのINP指標(2024年3月導入)により、アクセシビリティとパフォーマンスは「好み」から「法務・指標」の問題になった。
  • BtoBサイトは二正面作戦へ。「Linear/Stripe風」ダークSaaSデザインとベントグリッドはコモディティ化し差別化要因ではなくなった。次の課題は、編集的デザインや脱テンプレートによる「差別化」と、AI検索に引用されるための構造化・FAQ設計(AEO)という「AI可読性」の2つ。日本のBtoBでは見た目のインパクトより情報探索性・信頼性(実績ロゴ・認証)が引き続き効く。
  • Wixでの再現性は「大半が可能、ただし条件つき」。Wix Studio(2023年発表)のカスタムCSS・グラス効果・アニメーション機能・Velo(JavaScript)・iframe埋め込みを組み合わせれば、主要トレンドの見た目はほぼ再現できる。弱点はCore Web Vitals(良好サイト比率52%と競合比で劣後)、ページ全体と連動する本格3D、OS連動の自動ダークモード。第8章のマトリクスと第9章のレシピを参照。

第2章

注目トレンド深掘り

Apple「Liquid Glass」— 2025年最大のデザインイベント

発表
2025年6月9日、WWDC25
適用
iOS 26 / iPadOS 26 / macOS Tahoe / watchOS 26 / tvOS 26 / visionOS 26
ひとことで
背景を動的に反射・屈折させる「液体ガラス」質感の統一デザイン言語。iOS 7以来最大級の刷新

系譜としては、Windows Aero(2007)→iOS 7の半透明(2013)→Fluent Design(2017)→「Glassmorphism」命名(2020年、Michał Malewicz氏)→Liquid Glass(2025)という「デジタルガラス」表現の到達点。従来のすりガラス(ぼかし)と違い、リアルタイムレンダリングによるレンズ効果(光の屈折・集光)が特徴で、Webでも「より本物のガラスに近い」表現がCSS/WebGLで模索されている。Googleも2025年5月に弾性アニメーションとパーソナライズを掲げた「Material 3 Expressive」を発表しており、2大プラットフォームが揃って「表現性」に舵を切った年として記憶される。

ベントグリッド — デフォルト化とその先

日本の弁当箱に由来する名前のとおり、サイズの異なるカードを敷き詰めて情報の優先度を視覚化するレイアウト。Appleが2022〜2023年の製品ページ・キーノートで多用して一般化し、Vercel・Linear・FramerなどのSaaSに波及、2025〜2026年には「例外ではなくデフォルト」と言われるまで定着した。定着しきった結果、あえて枠からはみ出す「崩しベント」などの進化系が次の差別化として登場している。

「AIらしさ」への反動 — 質感と不完全さの復権

AI画像生成の普及で、紫〜青のグラデーション、金色がかったトーン、対称的すぎる構図といった「AIらしい見た目」がテンプレート化し、ユーザーに識別・忌避されるようになった。その反動として2025〜2026年に強まっているのが、ノイズ・グレイン(フィルム質感)、手描き風イラスト、コラージュ、非対称な構図など「人間の手仕事感」を演出する表現である。ネオブルータリズム(太い輪郭線・硬い影・生っぽいHTML感)も同じ文脈の対抗運動で、複数のトレンド予測が「2026年後半以降の中心的議論になる」と見ている。

技術面 — 演出がJSからCSSネイティブへ

スクロール連動アニメーションは、GSAP等のJSライブラリ依存から、CSS Scroll-driven Animations(animation-timeline: scroll() / view())によるブラウザネイティブ実装へ移行が進む(Chrome先行、Safari/Firefoxは追従中で完全な相互運用は途上)。バリアブルフォントも転送量を大幅に削減できる実用面からトレンドを超えて標準実務になった。「演出の実装コストとパフォーマンス負荷が下がった」ことが、モーション豊かなデザインの追い風になっている。

第II部 手法・システム面 — 見た目を支える(縛る)もの

第3章

デザインシステムとAI制作 — 均質化とその反動

  • デザイントークンの標準化 — Figmaの「Variables」(2023年)で実務浸透が進み、2025年10月にはW3C Design Tokens仕様がv1.0に到達(Adobe・Google・Microsoft・Figma等が支持)。色・余白・タイポをトークンとして一元管理し、ライト/ダークのモード切替まで扱うのが標準になった。日本ではデジタル庁デザインシステム(DADS)が代表例。
  • 「均質の海(Sea of Sameness)」問題 — Tailwind CSS/shadcn/uiとAIサイトビルダー(v0、Lovable等)の普及で、インディゴのグラデーション・角丸カード・予測可能な余白という「平均値のデザイン」が量産される状況が批判の的に。2025年8月にはTailwind作者Adam Wathan氏の「全ボタンをindigo-500にしたことを謝る」という投稿が大きな反響を呼んだ。カスタムテーマ・独自トークンへの回帰が対抗トレンド。
  • AIは制作の前提になった — Figmaの調査ではデザイナーの72%が生成AIをワークフローに活用。プロンプトからUIを生成するツール群が実務インフラ化し、デザイナーの役割は「手を動かす」から「ブランドに合うものを選び取るキュレーション」へ移行しつつある。さらにVercelの「Generative UI」やGoogleの「A2UI」など、対話から動的にUIを生成する仕組みも立ち上がっている。
第4章

「ルール」になった外部制約 — アクセシビリティと速度

制約時期デザインへの影響
WCAG 2.2 2023年10月 W3C勧告 フォーカス表示・認証方法など9つの達成基準を追加。コントラスト・フォーカスリングの設計が必須要件に
欧州アクセシビリティ法(EAA) 2025年6月28日 適用開始 EC・銀行・運輸等のサイト/アプリにアクセシビリティを義務化(EN 301 549 ≒ WCAG 2.1 AA)。2025年11月に仏で初訴訟、蘭でも執行開始——「努力目標」から「法的リスク」へ
Core Web Vitals / INP 2024年3月 INPがFIDを置換 全インタラクションの応答性を評価。重いJS演出・モーダル多用が敬遠され、軽量CSSアニメーションへの移行を後押し
prefers-reduced-motion 主要ブラウザ対応済み 前庭障害・てんかん等への配慮としてアニメーション抑制を実装するのがベストプラクティスに。「動きで魅せる」トレンドと対の関係

要するに、2025年以降の「良いデザイン」は派手さと配慮の両立が前提になった。Liquid Glass的なモーション豊かな表現を採るなら、reduced-motion対応・コントラスト確保・軽量実装をセットで設計する必要がある。

第III部 日本とBtoB

第5章

日本のWebデザイン — 文化的土台×海外トレンドのハイブリッド

  • 情報量・安心感重視の土台は不変 — 詳細説明・保証情報を厚く載せる「多階層・多要素型」の設計が日本のコーポレートサイトの基調。欧米の「余白と要点」型とは対照的で、BtoB製造業・金融・地方企業で顕著。2025年以降は、この情報量を維持したままAIに読ませやすく構造化する(見出し・箇条書き・FAQ化)方向にシフトしている。
  • イラスト・キャラクター文化 — キャラクターを前面に出す訴求は採用サイト・toCサービス・自治体で定着。生成AIによる制作コスト低下でむしろ活用が広がる可能性がある。
  • 和モダン・縦書きはニッチだが健在 — ギャラリーサイトMUUUUU.ORGには縦書き・縦組みカテゴリ(163件)があり、飲食・伝統工芸・和ブランドで根強い。ただし主流は横組み+グリッドで、話題の中心はベントグリッドやグラスモーフィズムなど海外発トレンド。
  • 国内制作会社の2025〜2026年共通項 — 「意味を持つカーソルUI」「グラスモーフィズム」「Bento UI」「文字主役のヒーロー」「ダークモード」「AIO/LLMO配慮の構造化」が複数社のトレンド記事で共通して挙げられ、2026年予測では手描き風・コラージュなどの「ハンドクラフト」系が新芽として登場。
  • 日本語Webフォント事情 — Google Fontsの日本語ファミリーは2018年の8件→2025年末66件に拡充。モリサワも低価格プラン(2025年)を追加。ただし日本語フォントは文字数ゆえに重く、サブセット化・ウェイト絞り込み・font-display設定などの最適化が実務必須のまま。
  • デジタル庁デザインシステム(DADS) — アクセシビリティ最優先の設計思想で継続更新中(2024年度グッドデザイン・ベスト100)。e-Gov等で採用が進み、官公庁サイトの標準化を主導。民間の意匠系トレンドとは別文脈の影響力を持つ。

第6章

BtoB・SaaSサイトのデザイン — コモディティ化と次の一手

「Linear/Stripe風」の完全コモディティ化

「ダーク基調+アクセントグラデーション+巨大タイポ+ベントグリッド」というLinear/Stripe/Vercel発の様式は、FramerテンプレートやAI生成ツールに組み込まれて完全にコモディティ化した。SaaS LPの定番構成——ヒーロー → 導入企業ロゴウォール → ベントグリッド型機能紹介 → 顧客事例 → CTA——も事実上の業界標準。ある調査ではProduct Hunt上位サイトの67%がベントグリッドを採用と報告される(ブログ推計値)。つまりこの様式は「やって当たり前、やっても差別化にならない」段階にある。

差別化の2路線

  • 編集的・脱テンプレート路線 — Notion・PostHog・Anthropicのように、雑誌的レイアウトや余白・タイポグラフィ中心のミニマルで「量産型SaaS」と距離を置く。AI企業風の装飾を削いだデザインもこの系譜。
  • あえて崩す路線 — ネオブルータリズムや手描き感で「人間がつくった感」を出す。2026年後半に向けた新芽。

変わらず効く「地味な基本」と新要件AEO

  • 信頼性の設計は不変 — ロゴウォール、導入社数などの数値、第三者認証(Pマーク・ISMS)、顧客事例。トレンドではなく必須要件。
  • 日本のBtoB評価は実用性重視 — トライベック「BtoBサイト調査2025」(有力197サイト・ユーザー7,700人)では三菱電機(FA)が3年連続総合1位。ナビゲーション・サイト内検索など情報探索性が最重要視され、欧米型の「見た目のインパクト」より実用が効く。ベイジも「クリエイティブは目的達成の手段」という立場で装飾を削いだ設計を提唱。
  • プロダクトの見せ方 — 静的スクリーンショット+デバイスモックから、ブラウザフレームでのアニメーション動画・操作可能な埋め込みデモへ移行中。
  • AEO(AI検索対応)が「見た目」と同格の要件に — 質問形式の見出し、冒頭での直接回答、FAQ・構造化データ(JSON-LD)の実装が、AI回答への引用可否を左右する。構造化データを欠くサイトはAI経由流入で不可視化されるとの実測報告もあり、2026年の「隠れた勝者」領域とされる。
  • フォーム(EFO) — 長い一覧型から一問一答のチャット型へ。BtoBでは商談・受注まで見据えた必須項目の絞り込みが重視される。

第IV部 Wix再現ガイド — トレンドをWixでどこまで作れるか

第7章

Wixの基礎知識 — 無印エディタとWix Studioの違い

最初に押さえるべきは、「どちらのエディタを使うか」で再現できる範囲が大きく変わることである。トレンド再現を狙うなら実質的にWix Studio一択と考えてよい。

項目Wixエディタ(無印)Wix Studio
位置づけ従来型の一般向けエディタ2023年発表の上位エディタ(Editor Xの後継。2025年1月に移行完了)。プロ・制作会社向け
カスタムCSS不可可(グローバルCSS+クラス単位のスタイル指定。2024年以降本格提供)
レスポンシブPC/モバイルの2種を個別調整既定3ブレイクポイント+最大6段階の独自定義、AIによる自動レスポンシブ化
アニメーション基本のスクロール/ホバー効果スクロールエフェクト・マウス追従パララックス+アニメーションライブラリ「Wix Interact」
グラス効果—「Apply glass effect」をノーコードで適用可
コード拡張Velo(JavaScript)利用可Velo+カスタムCSS+HTML埋め込み(iframe)

共通の注意点として、(1)無料プランはWixサブドメイン+Wix広告表示+外部解析ツール制限があり実務利用は有料プラン前提、(2)パフォーマンスはCore Web Vitals良好サイト比率52%との第三者調査があり、Duda(71%)・Squarespace(58%)に劣後(WordPress 38%よりは良い)。画像圧縮・フォント軽量化・アプリ追加の抑制など運用側の節制が重要になる。なお2026年にはAIエージェントによるサイト生成「Wix Harmony」の投入が報じられており、AI機能は急速に高度化している。

第9章

実装レシピ — 代表6パターン

レシピ1: ベントグリッド

Studioエディタでセクションにグリッドレイアウトを適用し、列を4分割・行を2分割、ギャップを全体で統一(12〜16px)。主役のカード1枚だけ2列×2行にスパンさせ、他は1マスに。カードの角丸(12〜16px)と背景色のトーンを揃えると「弁当箱」らしくまとまる。モバイルのブレイクポイントでは1〜2列に自動カスケードされるが、順序は手動で確認する。

レシピ2: グラスカード

まずStudioのInspectorから「Apply glass effect」を試す(ぼかし強度と背景色をノーコード調整)。細かく制御したい場合はコードパネルのグローバルCSSでカスタムクラスを作り、要素に割り当てる:

.glass-card {
  background: rgba(255, 255, 255, 0.10);
  backdrop-filter: blur(14px);
  -webkit-backdrop-filter: blur(14px); /* Safari用 */
  border: 1px solid rgba(255, 255, 255, 0.25);
  border-radius: 16px;
}

背景に色の変化(グラデーションや写真)がないとガラス感が出ないので、必ず賑やかな背景の上に置くこと。

レシピ3: オーロラ背景+グレイン

確実で軽いのは、FigmaなどでオーロラグラデーションをWebP画像(1枚・200KB以下目安)に書き出して背景に敷く方法。CSSでやるならグローバルCSSで:

.aurora-bg {
  background:
    radial-gradient(ellipse at 20% 30%, #35d0a0 0%, transparent 55%),
    radial-gradient(ellipse at 80% 20%, #3e7df0 0%, transparent 55%),
    radial-gradient(ellipse at 60% 85%, #b44fd8 0%, transparent 55%),
    #101825;
}

グレインは小さなノイズ画像(PNG/SVG)を半透明で重ねる。「AIっぽい紫グラデ」に寄りすぎないよう、ブランドカラー由来の配色に変えるのがポイント。

レシピ4: バリアブルフォントの可変軸

WOFF2形式のバリアブルフォントをアップロード後、グローバルCSSで軸を直接指定する:

.hero-title {
  font-variation-settings: "wght" 850, "wdth" 110;
}

日本語フォントは容量が大きいため、見出し用にサブセット化(使用する文字だけに絞る)してから使う。本文は標準提供フォントに任せるのが速度面で安全。

レシピ5: Spline(3D)の埋め込み

Spline側でシーンを「Export → Viewer/Embed」で公開URL化し、WixのHTML iframe要素にURLを設定(Spline公式にWix向け手順あり)。注意点は2つ: (1)iframeサンドボックス内で動くため、ページのスクロールや他要素と連動させる演出は不可。(2)3Dは読み込みが重いので、ファーストビュー直下には置かずINP/LCPへの影響を確認する。

レシピ6: FAQ構造化データ(AEO対応)

ページのSEO設定 →「構造化データマークアップ」からJSON-LDを追加する(7,000文字以内):

{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "導入までの期間はどのくらいですか?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "標準プランで約2週間です。..."
    }
  }]
}

ページ本文側も「質問形式の見出し+冒頭で直接回答」の構造に揃えると、AI検索(AI Overviews・ChatGPT等)に引用されやすくなる。BtoBサイトではこれが第6章で述べた「AI可読性」対応の第一歩になる。

第10章

総括 — 2026年にサイトを作るなら

  • 「定着済み」は土台として使い、差別化を上乗せする。ベントグリッド・ダークモード・マイクロインタラクションはもう「当たり前」で、やっても目立たないがやらないと古く見える。差別化は質感(グレイン・手描き感)、編集的レイアウト、ブランド固有の色・書体で作る。トレンドの紫グラデ・indigo系の配色は「AI量産感」と受け取られるリスクがあるため避けるのが無難。
  • 派手さは配慮とセットで。モーション豊かな演出を入れるなら、コントラスト・フォーカス表示・モーション抑制・表示速度(INP)を同時に設計する。EAA施行以降、アクセシビリティは欧州向けビジネスでは法的要件でもある。
  • BtoBは「見た目の様式」より「信頼と可読性」。ロゴウォール・数字・事例・探しやすいナビという基本を固めた上で、AI検索に引用される構造化(質問見出し+直接回答+JSON-LD)を新しい必須要件として組み込む。
  • Wixで作るなら:Studio+3つの逃げ道。グラス効果・グリッド・Interactなどの標準機能でトレンドの8割は形になり、残りはカスタムCSS/Velo/iframe埋め込みで補える。ただしパフォーマンスが弱点なので、画像・フォント・埋め込みの「足し算」を節制し、3Dや重い演出は本当に必要な1箇所に絞るのが成功パターン。
